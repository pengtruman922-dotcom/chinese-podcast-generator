# 中文播客生成应用 PRD

## 1. 产品概述

### 1.1 产品定位
将英文音视频自动转写为中文语音播客的工具，面向AI资讯类播客场景。

### 1.2 核心价值
- 自动化完成英文ASR → 中文翻译 → 中文语音合成全流程
- 保证专业术语、人名和公司名的处理一致性
- 支持说话人识别与音色映射

## 2. 技术架构

### 2.1 实现方式
- **后端**：Python脚本串联流程（本地运行，后续部署阿里云）
- **前端**：React + Vite + TypeScript + Tailwind（已有未联调代码）
- **存储**：本地文件存储，支持在线试听与公开链接下载

## 3. 核心流程

### 3.1 流程概览
```
用户上传音频/视频 → ASR英文识别 → AI翻译与格式化 → TTS中文合成 → 在线试听/下载
```

### 3.2 详细流程

#### 3.2.1 上传阶段
- **输入**：音频文件（mp3、wav）或视频文件（mp4、mov等，仅提取音轨）
- **输出**：本地存储路径，生成任务ID

#### 3.2.2 ASR识别阶段
- **服务商**：火山引擎 OpenSpeech BigASR
- **配置**：
  - 模型版本：`version=400`（固定）
  - 分句标识：`timestamps=10s`（固定）
  - 说话人识别：默认开启（`enable_speaker_info=true`）
  - 性别检测：默认开启（`enable_gender_detection=true`）
  - 热词支持：文本框输入，逗号分隔
- **接口模式**：submit → query 轮询
  - 轮询间隔：5-60s可调（默认10s）
  - 超时时间：6000s（固定）
- **输出格式**：
```json
{
  "result": {
    "utterances": [
      {
        "text": "英文识别文本",
        "start_time": 0,
        "end_time": 1000,
        "speaker": "0",
        "additions": {
          "speaker": "0",
          "gender": "male"
        }
      }
    ]
  }
}
```

#### 3.2.3 翻译与格式化阶段
- **服务商**：阿里灵积 DashScope（兼容 OpenAI 接口）
- **配置**：
  - 模型名称：可编辑（默认 `deepseek-v3`）
  - API地址：可编辑
  - 系统提示词：可编辑
  - 用户提示词：可编辑，默认加载ASR识别的文本
- **翻译规则**：
  - AI领域专业术语保留英文（multi-clouding、Cloud Code、API、token等）
  - 公司名称保留英文（Anthropic、OpenAI等）
  - 人名保留英文（Alex、Cat等）
  - 保持口语化风格，适合语音播报
- **说话人合并规则**：
  - [Speaker 1] → "发言者1"
  - [Speaker 2/3/4...] → 统一为"发言者2"
  - 同一说话人连续多段内容合并为一段
- **输出格式**（严格JSON数组）：
```json
[
  {
    "speaker": "发言者1",
    "text": "翻译后的中文文本"
  },
  {
    "speaker": "发言者2",
    "text": "翻译后的中文文本"
  }
]
```

#### 3.2.4 TTS合成阶段
- **服务商**：火山引擎 TTS
- **配置**：
  - 资源版本：1.0/2.0 可选（默认2.0）
  - 并发上限：1-10可调（默认5）
  - 同人停顿：100ms（固定）
  - 换人停顿：300ms（固定）
- **音色映射**：
  - 发言者1默认：`zh_female_xiaohe_uranus_bigtts`（小何-通用女）
  - 发言者2默认：`zh_male_taocheng_uranus_bigtts`（小天-通用男）
  - 注意：不同资源ID（1.0/2.0）的音色列表不同
- **处理流程**：
  - 并发调用TTS接口获取PCM音频片段
  - 按对话顺序拼接，同人/换人插入对应停顿
  - 输出24kHz单声道PCM wav文件
- **输出**：result_HHMMSS.wav

#### 3.2.5 发布与下载
- **存储**：本地文件系统
- **访问方式**：
  - 在线试听（前端Audio组件）
  - 公开链接下载（wav文件）

## 4. 数据结构定义

### 4.1 任务状态机
```
Uploaded          → 文件已上传
ASR_Submitted     → ASR任务已提交
ASR_Processing    → ASR识别中
ASR_Done          → ASR完成
Translate_Processing → 翻译中
Translate_Done    → 翻译完成
TTS_Processing    → TTS合成中
TTS_Done          → TTS完成
Publish_Done      → 发布完成（可下载）
Failed            → 任务失败
```

## 6. 非功能需求

### 6.1 性能要求
- TTS并发合成支持1-10路并发（默认5）
- Session复用降低网络开销
- 1分钟音频处理总耗时目标：< 3分钟

### 6.2 容错与降级
- ASR轮询超时处理：返回接口完整错误信息
- TTS流式片段解析失败：跳过该片段并记录日志
- 翻译接口网络异常：重试机制（可选）
- 说话人标注缺失：自动降级为单人播报

### 6.3 安全与合规
- API密钥存储：环境变量或服务端配置文件
- 禁止代码硬编码密钥
- 上传文件类型校验
- 无IP仿音限制

### 6.4 日志与错误处理
- 不需要结构化日志与错误码表
- 直接返回接口完整错误信息
- 关键阶段记录进度（上传、ASR、翻译、TTS、完成）

## 10. 风险与规避

### 10.1 技术风险
| 风险项 | 影响 | 规避措施 |
|--------|------|----------|
| 第三方API限流 | 任务失败 | 重试机制、排队策略 |
| ASR说话人识别不稳定 | 音色映射错误 | 自动降级为单人播报 |
| 长音频内存占用高 | 程序崩溃 | 分段处理、增量写入 |
| 翻译术语不准确 | 播客质量差 | 热词支持、提示词优化 |

### 10.2 业务风险
| 风险项 | 影响 | 规避措施 |
|--------|------|----------|
| 密钥泄露 | 安全问题 | 环境变量管理、禁止硬编码 |
| 音频版权问题 | 合规风险 | 用户协议声明、仅供个人使用 |
